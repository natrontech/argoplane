ArgoPlane Design System

Minimal pixel art meets modern developer tooling. One design system for all ArgoPlane surfaces: ArgoCD extensions, self-service portal, documentation, and landing page.

PHILOSOPHY

Pixel-crisp, not retro

Everything on a 4px grid. Crisp 1px borders. 8x8 status squares. That satisfying feeling of something perfectly aligned, without the 8-bit nostalgia.

Pastel palette

Warm grays, soft status colors, and a pastel orange accent. Easy on the eyes during long debugging sessions. Pops enough to convey meaning.

Flat and honest

No shadows, no gradients, no animations over 150ms. Information density over visual flair. Every pixel earns its place.

One system, all surfaces

Same tokens, same components whether you're inside ArgoCD, on the docs site, or on the landing page. CSS custom properties as the shared foundation.

HOW TO USE

Full bundle

<link rel="stylesheet" href="design-system/argoplane.css">

Individual layers

<link rel="stylesheet" href="design-system/tokens.css">      <!-- always required -->
<link rel="stylesheet" href="design-system/base.css">        <!-- reset + typography -->
<link rel="stylesheet" href="design-system/components.css">   <!-- .ap-* component classes -->
<link rel="stylesheet" href="design-system/utilities.css">    <!-- .ap-flex, .ap-p-4, etc. -->

React (ArgoCD extensions)

Import from @argoplane/shared which re-exports tokens as TypeScript constants and provides React components that use the same design system. 

import { colors, spacing, StatusBadge, Button } from '@argoplane/shared';

Dark theme

Add data-theme="dark" or class theme-dark to any parent element. All semantic color tokens automatically switch. 

<body data-theme="dark">  <!-- everything inside uses dark palette -->

ANTI-PATTERNS
No rounded pills or circles. Max border-radius: 4px.
No box shadows. Pixel art is flat.
No gradients. Flat colors only.
No animations longer than 150ms (except loading indicator).
No font sizes above 20px. Keep it compact.
No emoji in UI. Use pixel squares and symbols.
No hardcoded colors. Always use tokens.
